## 포워드 프록시와 리버스 프록시의 개념을 설명할 수 있나요?  

포워드 프록시는 클라이언트를 대리하는 프록시입니다. 클라이언트가 서버로 요청을 보낼 때, 포워드 프록시가 요청을 받아 서버로 전달합니다. 서버 입장에서는 포워드 프록시의 IP만 확인이 가능합니다. IP를 우회하거나, 내부망에서 특정 사이트로의 접근을 차단하는 역할을 수행할 수 있습니다.

리버스 프록시는 서버를 대리하는 프록시입니다. 서버 앞단에 위치합니다. 클라이언트의 요청을 리버스 프록시가 받고, 내부의 여러 애플리케이션 서버로 전달합니다. 로드 밸런싱 용도로 사용하거나, 외부의 공격을 일차적으로 걸러줍니다.

## Java 에서 객체의 동일성을 어떻게 비교할까요?

객체를 비교하는 방법에는 == 연산자와 .equals() 메서드가 있습니다.

== 연산자는 물리적 동일성을 비교합니다. 두 참조 변수가 힙 메모리에서 정확히 동일한 객체를 가리키고 있는지, 메모리 주소값을 비교하는 것입니다.

반면 Object 클래스의 .equals() 메서드는, == 연산자와 똑같이 작동합니다. 그러나 String, Integer와 같은 Java의 래퍼 클래스들은 .equals() 메서드를 오버라이딩 하여, 값이 같은지를 확인합니다.

## jvm 메모리란 ?  
jvm은 자바 가상 머신, 즉 자바 프로그램을 실행하기 위한 가상의 소프트웨어입니다.Java의 철학인 한번 작성하면 어디서든 실행된다. 를 구현하기 위해 존재합니다. Java코드를 컴파일하면, Java 바이트코드가 생성됩니다. 그러면 JVM이 이를 기계어로 번역시킵니다.
이 외에도 메모리 관리와 보안 같은 기능도 jvm이 합니다. 

Java를 실행하기 위해 OS로부터 할당받는 메모리 공간입니다. 모든 스레드가 공유하는 영역과 스레드별 영역으로 나뉩니다. 공유 영역에는 힙, 메소드 영역이 있습니다.

스레드별로는 스택과 PC 레지스터와 같은 영역을 지닙니다.
